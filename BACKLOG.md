## 🏗️ 4. 프로젝트 결정 사항 (Project Decisions)

*이 영역은 아키텍처/기획의 중요한 결정 사항이 변경될 때마다 업데이트하여, 코딩 전 에이전트가 참고하는 컨텍스트 주입기로 사용합니다.*

- **모바일 퍼스트 레이아웃**: 모든 화면은 최대 폭 480px의 컨테이너(`Layout.tsx`) 내부에서 렌더링됨. 어드민 페이지 역시 모바일 환경에서도 제어 가능하도록 반응형이 적용됨.
- **스타일링 원칙**: Tailwind 등 유틸리티 프레임워크를 배제하고, CSS Variables 기반 Vanilla CSS 사용. 우아하고 따뜻한 톤 유지.
- **동적 라우팅**: 개별 청첩장은 `/:invitationSlug` URL 패턴을 사용하며, 해당 파라미터 기준으로 DB 패치 수행.
- **백로그 포맷**: Markdown Table이 아닌 계층형 리스트 방식을 채택하여 상태(대기/진행/완료) 별로 구분 관리 (Git 충돌 방지 목적).
