# AGENTS.md (AI 행동 경계 협약)

이 문서는 본 프로젝트에 참여하는 모든 AI 에이전트(Agy, Claude, Gemini 등)가 **반드시 준수해야 하는 행동의 경계와 제약사항**을 명시합니다. 에이전트는 작업을 시작하기 전 이 문서를 숙지하고, 절대적으로 기준을 엄수해야 합니다.

## 1. AI 행동 권한표 (Permission Boundary)

| 행동 | AI 자율 실행 | 사람 승인 필요 | 비고 |
|---|---|---|---|
| **라이브러리(의존성) 추가** | ✅ (자유) | | 단, 설치 후 텍스트 보고 및 PR Comment에 반드시 명시할 것 |
| **로컬 브랜치 정리** | ✅ | | |
| **문서 오탈자 수정** | ✅ | | |
| **새로운 브랜치 생성 및 작업** | ✅ | | 모델명/에이전트명(claude, agy 등) 브랜치 이름 사용 **절대 금지**. 업무(태스크)에 맞는 적절한 이름 생성 필수 |
| **PR 생성 (Pull Request)** | ✅ | | |
| **`main` 브랜치 직접 Push** | | ✅ (절대 금지) | 어떠한 경우에도 `main` 직접 Push 불가 |
| **PR 머지 (Merge)** | | ✅ (절대 금지) | **류대성(기획/QA)**님만이 PR을 머지할 권한을 가짐 |

## 2. 브랜치 전략 및 작업 플로우 (Strict Git Workflow)
AI가 작업을 수행할 때는 **말하지 않아도 무조건 아래의 순서를 지켜야 합니다.**
1. **`main` 현행화**: 로컬의 `main` 브랜치를 최신 상태로 Pull (Sync) 합니다.
2. **작업 브랜치 생성**: 태스크의 목적을 명확히 알 수 있는 시맨틱 네이밍(예: `feat/add-kakao-pay`, `fix/gallery-stagger`)으로 브랜치를 생성합니다. (에이전트 이름 금지)
3. **작업 진행 및 커밋**: Conventional Commits 규칙 준수.
4. **PR 생성**: 리뷰어가 파악하기 쉽도록 작업 내역, 추가된 라이브러리를 PR Comment로 명시합니다.
5. **대기**: 머지는 사람(류대성 님)이 직접 수행하므로 대기합니다.

## 3. 에러 발생 시 반복 제한 (Runbook)
*   **에러 3회 제한 규칙**: AI는 빌드 실패, DB 연동 오류, 테스트 실패 등 **같은 종류의 에러가 3번 이상 반복되면 모든 시도를 즉각 멈춥니다.**
*   절대 접근법을 무작위로 바꾸어 토큰과 자원을 낭비하지 말고, 즉각 사람에게 현재 상태와 에러 로그를 보고한 뒤 지시를 대기합니다.

## 4. 완료 보고 기준
*   AI가 "작업을 완료했습니다"라고 보고할 때는, 반드시 **어떤 기준(로컬 테스트 통과, 문서 대조 일치, 컴파일 성공 등)으로 완료를 판단했는지 근거를 함께 제시**해야 합니다.
